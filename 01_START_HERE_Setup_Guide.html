2. 2.5D Layer Decomposition (Vector Guidelines)

To prepare files for this workflow, you must perform a Multi-depth Vector Segmentation (or 2.5D Layer Decomposition). You are converting a flat 2D graphic into independent, stacked profiles. The goal is: One clean, closed vector per height layer exported as an SVG.

⚠️ Critical Vector Health Rules

Autodesk Fusion performs heavy mathematical offsets to create the interlocking features. It will fail if your vectors are messy or left hollow.

  • Patch Underlying Holes: When you move upper-level details to their own layer, you MUST patch the hole they leave behind in the lower layer. Release compound paths and delete inner cutouts so the underlying shape is completely solid. The engine will automatically carve out the exact pocket needed.
  • Closed Paths Only: Every single shape MUST be a perfectly closed path. No open lines.
  • Avoid Node Clustering: Select your artwork and use Object > Path > Simplify to remove excess anchor points.
  • No Sharp Corners: Round off micro-sharp corners. Autodesk Fusion cannot calculate offsets for overlapping tight vertices.
  • 1:1 Scale: Draw exactly at scale (line-on-line). Do not manually offset paths to fake tolerances.
  • Expand Everything: Outline all text (Type > Create Outlines) and expand all strokes/appearances.

3. Layer Naming Syntax

Every layer meant for export must begin with lvl=. We use double underscores (__) to separate parameters.

lvl=h4__sem=CapTop__rol=insert__par=Base__col=Black__hex=000000
Parameter Meaning Example
lvl Z-Height Layer (Must be first) lvl=h2, lvl=h3+
sem Semantic Part Name sem=MainBody
rol Physical Role / Behavior rol=insert
par Parent Component (Optional) par=MainBody
col Autodesk Fusion Material Name col=MatteWhite
hex Appearance Color Code hex=FFFFFF

4. Role Dictionary

The rol parameter dictates exactly how the Autodesk Fusion engine processes boolean cuts and starting heights.

  • base: The foundation. Extrudes from 0mm up to its assigned level.
  • core: Stacks perfectly flush on top of its parent. No boolean cuts.
  • insert: Sinks into the parent, generating a stepped cutting tool with tight snap-fit tolerances.
  • detail: Mechanically identical to insert, but uses a micro-offset (0.05mm) to prevent failure on tiny text.
  • cover: Slides over the base. Ignored parent height, floating at 2.0mm and carving upward with loose slide-fit tolerances.
  • pillar: Anchors directly to 0mm and punches upward through any intermediate layers to prevent floating geometry.
  • drop: Anchors to the parent's top surface, but punches downward through anything beneath it.
